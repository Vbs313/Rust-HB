> 创建日期: 2026-05-23
> 状态: ✅ 文件已修改，待编译部署验证
> 前置: 2026-05-23-0027 异步架构重构完成

## 动机

Rust 侧通过 `CreateRemoteThread` + `ReadProcessMemory` 外部扫描 Mono 结构时遇到瓶颈：

| 问题 | 原因 |
|------|------|
| MonoClass.name 偏移不确定 | 尝试了 4 种偏移（0x08/0x0C/0x10/0x04）均不符合标准布局 |
| class_cache 找不到 | Assembly-CSharp 映像前 4KB 没有 GPtrArray/GHashTable |
| `mono_class_from_name` 不可调 | ACG/CIG 阻止多参数 shellcode 执行 |
| Hearthbuddy.exe 偏移提取失败 | Bots 路径问题导致启动崩溃 |

## 方案

BepInEx 插件已在游戏进程内运行，**ACG 管不到进程内的 C# 代码**。直接通过 `Type.TypeHandle.Value` 拿到 MonoClass 指针，无需外部内存扫描。

## 改动

### 文件: `ipc-plugin/RustIPCPlugin.cs`

**新增 1 个 `using`：**
```
using System.Runtime.InteropServices;  // Marshal.Copy 需要
```

**IpcMessage 增加 `args` 字段：**
```csharp
public class IpcMessage { public string type; public uint seq; public string args; }
```

**Dispatch 增加 2 个 case：**
- `DumpClass` — 传入类名，返回 MonoClass 结构体前 128 字节 hex dump + 可能的指针字段
- `EnumerateClasses` — 列出 Assembly-CSharp 所有公开类的 MonoClass 地址

### 新增方法

```csharp
HandleDumpClass(uint seq, string className)
  → Type.GetType(name + ", Assembly-CSharp")
  → type.TypeHandle.Value  // = MonoClass*
  → Marshal.Copy(raw, 0, 128)
  → 返回 hex + 指针猜测

HandleEnumerateClasses(uint seq)
  → AppDomain.CurrentDomain.GetAssemblies()
  → Assembly-CSharp 所有公开类的 FullName + MonoClass 地址
```

## 工作原理

```
Rust IPC Client                    BepInEx Plugin (进程内)
    │                                      │
    ├─ {"type":"DumpClass","args":"GameState"}
    │                                      ├─ Type.GetType("GameState")
    │                                      ├─ type.TypeHandle.Value → MonoClass*
    │                                      ├─ Marshal.Copy → 128 bytes raw
    │                                      └─ 返回 hex dump
    ├─ {"type":"EnumerateClasses"}
    │                                      ├─ 枚举 Assembly-CSharp 类型
    │                                      └─ 返回 FullName + 地址列表
```
