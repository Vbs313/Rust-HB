### 关键修复模式

为了避开 `windows` crate v0.58 的 API 兼容问题，`core`、`process-mgr`、`input-sim` 三个 crate 采用 **raw FFI** 直接声明 Win32 DLL 函数签名：

```rust
#[link(name = "kernel32")]
extern "system" {
    fn ReadProcessMemory(...) -> i32;
    fn WriteProcessMemory(...) -> i32;
}
```

这种方式提供：
- 零依赖编译（不需要 windows crate）
- 完全的控制和可预测性
- 与 C# 版 P/Invoke 的 1:1 对应

## 五、技术决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| Win32 API | **raw FFI** 而非 `windows` crate | 避免版本兼容问题，编译快 100 倍 |
| 错误类型 | `thiserror` + `anyhow` | 与 C# 异常等效，减少样板代码 |
| 异步运行时 | `tokio` | 替代 C# Coroutine 系统 |
| 卡牌生成 | `build.rs` 构建脚本 | 从 C# CardDB 源码自动生成 |
| Mono 偏移 | 运行时扫描而非硬编码 | 适应炉石更新 |
| 日志 | `tracing` + `env-filter` | 替代 log4net |
