# 逆向 Mono 运行时进展

## 已确认

### 工具链

1. **CreateRemoteThread 可用于调用 Mono PE 导出函数**（未阻塞 ACG，因为代码在合法 PE 模块中）
2. **可调用的函数**：`mono_get_root_domain()`、`mono_get_corlib()`（0 参数 cdecl 可用）
3. **不可调用的函数**：`mono_class_from_name()`（3 参数，需 shellcode 被 ACG 阻止）

### 内存锚点

| 项目                      | 当前实例地址 | 说明                               |
| ------------------------- | ------------ | ---------------------------------- |
| mono.dll                  | 0x777d0000   | PE 模块基址                        |
| Root Domain               | 0x09d22e70   | 通过 `mono_get_root_domain()` 获取 |
| corlib MonoImage          | 0x0969ea60   | 通过 `mono_get_corlib()` 获取      |
| Assembly-CSharp MonoImage | 0x0685f5d4   | 通过内存扫描 + 名称验证            |

### 结构偏移确认

| 结构      | 偏移  | 字段              | 验证方式                         |
| --------- | ----- | ----------------- | -------------------------------- |
| MonoImage | +0x1C | name (char\*)     | 验证 corlib/Assembly-CSharp 名称 |
| MonoImage | +0x20 | filename (char\*) | 验证 mscorlib.dll                |
| MonoImage | +0x28 | version (char\*)  | 验证 "v4.0.30319"                |

### 类名字符串地址

| 类名       | 地址       |
| ---------- | ---------- |
| SceneMgr   | 0x02b044cc |
| GameState  | 0x02b8ac18 |
| Entity     | 0x02b43918 |
| GameEntity | 0x02bacdb8 |

## 已发现但未解明

1. **MonoClass.name 偏移**：游戏类名字符串被引用，但引用点所在结构不包含 namespace 和 image 指针，因此不是标准 MonoClass
2. **class_cache 位置**：扫描 Assembly-CSharp 映像前 4KB 未找到 GPtrArray 或 GHashTable
3. **堆实例扫描**：`class_cache` 遍历返回 0

## 下一步建议

### 方案 A：偏移提取（原计划）

修复 Hearthbuddy.exe 启动崩溃（`E:\Next\Bots\` 路径问题），从运行中的 HB 提取 36 个偏移值：

```
mkdir E:\Next\Bots
copy E:\Next\HB3.1.2\Bots\* E:\Next\Bots\ /s
```

### 方案 B：堆扫描找 MonoClass（近期的逆向成果）

搜索 GC 堆中所有指向 Assembly-CSharp 映像地址 (0x0685f5d4) 的 u32 指针，那些结构全是 MonoClass 实例。
