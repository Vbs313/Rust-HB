# 路线 A 实施完成—BepInEx IPC 插件

## 实施内容

### 新建: `E:/Next/ipc-plugin/`
- **`RustIPCPlugin.cs`** (~550行) — BepInEx 插件
  - 命名管道服务端 `\\.\pipe\Hearthbuddy_IPC`
  - 接收 GetGameState → 通过反射调用 Hearthstone C# API 读取游戏状态 → 返回 JSON
  - 接收 Ping → 返回 Pong（心跳检测）
  - 接收 PerformAction → 执行操作（EndTurn / PlayCard / AttackWithMinion / UseHeroPower）
  - 运行时解析 Assembly-CSharp 类型（无需编译时引用）
- **`RustIPCPlugin.csproj`** — .NET Framework 4.7.2 项目文件
- **编译产物** `bin/RustIPCPlugin.dll` → 已安装到 `BepInEx/plugins/`

### 相关配置
- `HB3.1.2/HsMod/doorstop_config.ini` → `enabled=true` 启用 Unity Doorstop 注入
- `HB3.1.2/HsMod/BepInEx/plugins/RustIPCPlugin.dll` ✅ 已部署

### Rust 侧（无需修改）
- `crates/app/src/main.rs` — 已支持 IPC 模式（先连 IPC，失败则离线演示）
- `crates/ipc/src/lib.rs` — 协议匹配，9 个序列化测试已通过
- AI 引擎 `DefaultRoutine.our_turn_logic()` — 直接使用从 IPC 读取的真实局面

## 架构

```
Hearthstone.exe (Unity)
  └─ BepInEx (Doorstop 注入)
       └─ RustIPCPlugin.dll ← 命名管道服务器
            │
hb-app (Rust) ← IPC 客户端 → 获得 GameState → Playfield → AI 引擎
            │
hb-input-sim (鼠标/键盘) → 执行操作（出牌/攻击）
```

## 运行步骤

1. 启动 Hearthstone.exe（战网启动）
2. 启动 Hearthbuddy.exe（它会自动加载 BepInEx + 插件）
3. `cargo run --target i686-pc-windows-msvc -p hb-app`（Rust 端连接 IPC）
